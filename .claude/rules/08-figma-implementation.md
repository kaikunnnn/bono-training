# Figmaデザイン実装の必須手順

## 実装前（必須）

1. **ブロック/コンポーネント単位で実測する**: 対象のFigmaノードを `get_design_context`（Figma MCP）で取得し、実測値（色・余白・フォントサイズ・角丸・gap）を**仕様書ファイルに書き出す**
   - 保存先: issueフォルダに `〜_Figma仕様.md` として永続化する
   - ❌ セッション限りのscratchpadに置かない（次のセッションで参照できなくなる）

2. **トークン対応表を先に作る**: Figmaの生値 → デザインシステムトークンの対応表を仕様書に含める

   | Figma実測値 | 使用トークン | 備考 |
   |---|---|---|
   | `#d9d9d9` | `border-input` | 近似置換（明記する） |
   | `#102720` | `bg-btn-primary` | 完全一致 |

   - 近似置換は**表に明記**する
   - 対応トークンが無い値は `colors_and_type.css` にトークンを追加してから使う（生hex/rgbaのJSX直書き禁止）

3. **Figmaの値を勝手に近似で丸めない**: 判断に迷う差（例: 独自色を `bg-muted` に丸める）は「意図的にFigmaから変えた箇所」として報告し、ユーザーの判断を仰ぐ

## アニメーション

- `get_motion_context` でキーフレームが取れる（取れれば数値・イージング付きで実装する）
- 取れないノードはスクショ + 言語仕様を書いてから自作する

## 実装後（コミット前に必ず実施）

1. **生hex grep（機械検証)**: 変更ファイルに対して `grep -nE '#[0-9a-f]{3,8}|rgba?\(' ` を実行し、新規の生値が無いことを確認
2. **スクショ突合**: ブラウザスクショとFigmaスクショを並べて比較する
3. **修正時は3列突合表を出す**: 「Figma値 / 修正前 / 修正後」の表で差分を提示する

## ノードがAPIから見えない場合

ブランチ未マージ / 別ファイル / 同期ラグを疑う。**適当に済まさずユーザーに確認する**（⌘⌥Sで版保存を依頼する）。

```
// ❌ 間違い: ノードが取れないので近いスクショから雰囲気で実装
// ✅ 正しい: 「対象ノードがAPIから見えません。Figmaで⌘⌥Sで版を保存してもらえますか？」と確認
```
