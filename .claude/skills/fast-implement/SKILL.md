# Fast Implementation Mode

## 概要
確認ステップを最小限にして、高速で実装を進めるモード。

## 呼び出し方
- `/fast-implement [タスク内容]`
- 「確認なしで実装して」
- 「fast modeで」

---

## 実装ルール

### やること
1. タスクを受け取ったら即座に分析開始
2. 必要なファイルを読み込み
3. 変更を実装
4. 完了後にまとめて報告

### やらないこと
- 「〜してもいいですか？」と聞かない
- 「〜で進めますがよろしいですか？」と聞かない
- 実装前の詳細な計画確認を求めない
- 各ステップごとの承認を求めない

### 報告形式
実装完了後、以下の形式で報告：

```
## 完了

### 変更ファイル
- `path/to/file1.tsx` - 変更内容
- `path/to/file2.ts` - 変更内容

### 変更概要
[簡潔な説明]

### 確認方法
[動作確認の手順]
```

---

## 例外（必ず確認する）

以下の操作は確認を必須とする：

1. **本番環境への影響**
   - デプロイコマンド
   - 本番DBへの操作
   - Webhook/API設定の変更

2. **破壊的操作**
   - ファイル/フォルダの削除
   - データベースのデータ削除
   - git reset --hard などの取り消し不可操作

3. **セキュリティ関連**
   - 環境変数の変更
   - シークレット/トークンの操作
   - 認証設定の変更

4. **大規模リファクタリング**
   - 10ファイル以上の変更
   - 既存APIの破壊的変更

---

## エラー対応

エラーが発生した場合：
1. 自動で原因を分析
2. 修正を試みる
3. 3回試行しても解決しない場合のみ報告
