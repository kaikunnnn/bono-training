トレーニング：お題だけが並んでいるテストページです
みんなの掲示板：テスト用です。消します
設定
ログアウト

▼ 追加したい

□ ナレッジ
↑の質問とかナレッジでよく出てくるな〜ってやつをコースなどにする前に１つまとめた記事を書いてみるものを公開するものを想定しています。これはメンバー以外も見れます。つまりこれをSNSでシェアすることでBASEってこういうことやれるんだ！とかBONOを知るきっかけにしていきたいです。

□ Slackからどう持ってくる？
基本的には以下
- Slackで質問を受ける。その回答をこのページに書く→Slackで共有→追加質問などがあれば追記する。です。

- ちなみにゆくゆく「掲示板」みたいなのでメンバーのみ見れて書き込みできる場所を作ろうかなと思っているけどおそことの棲み分けも考えられるなら考えたい（難しいなら一旦考えなくていい

## フィードバック機能

**ステータス**: 実装計画確定 ✅

### カテゴリ（確定）

| カテゴリ | 説明 |
|---------|------|
| ポートフォリオ | ポートフォリオ全体のフィードバック |
| ユーザー価値設計 | ユーザー課題の特定、解決策の妥当性 |
| UIスタイル | 基本的なUIパターン、スタイルの正しさ |
| キャリア | キャリア相談 |

### 一覧ページ `/feedback`

**レイアウト**: 質問ページと同じ幅・レスポンシブ

**ページタイトル**: 共通コンポーネント化（質問ページから抽出）
- ラベル: "Feedback"
- タイトル: "みんなのフィードバック"
- ボタン: 状態により出し分け

**ボタン出し分けロジック**:
| 状態 | 表示 |
|------|------|
| 未ログイン | ログイン促すボタン |
| ログイン済み（グロースプラン以外） | 「プランの変更が必要です」+ 「プランを変更する」ボタン |
| グロースプラン | 「フィードバックを依頼する」ボタン |

**フィードバックカード**:
- テイスト: LessonCardを参考
- タイトル（必須）
- カテゴリ（必須）
- 対象アウトプット/コース名（任意、なければ非表示）
- 概要文（最大2行、必須）

### 詳細ページ `/feedback/:slug`

**構成（上から順に）**:

| # | セクション | 内容 | アクセス |
|---|-----------|------|----------|
| 1 | ヘッダー | タイトル・カテゴリ・対象アウトプット・日付 | 全員 |
| 2 | 動画 | Vimeo埋め込み（ある場合のみ、ブログ詳細のサムネ位置） | **課金者限定**（VideoSection使用） |
| 3 | 依頼内容 | FigmaURL（リンクのみ）・見てほしいポイント・依頼文 | 全員 |
| 4 | 本文 | Kaiのフィードバック（リッチテキスト、ブログ風） | **課金者限定** |

### Sanity Schema設計

**feedbackCategory**:
- title: string
- slug: slug

**feedback**:
- title: string（必須）
- slug: slug（必須）
- category: reference → feedbackCategory（必須）
- targetOutput: string（任意、対象コース/アウトプット名）
- excerpt: text（概要、最大2行分）
- videoUrl: url（任意、Vimeo）
- figmaUrl: url（任意）
- requestPoints: text（見てほしいポイント）
- requestContent: blockContent（依頼内容リッチテキスト）
- feedbackContent: blockContent（本文リッチテキスト）
- publishedAt: datetime

### 実装で使用する既存コンポーネント

- `VideoSection`: 動画表示 + アクセス制限（src/components/article/VideoSection.tsx）
- `PremiumVideoLock`: 動画ロック表示（src/components/premium/PremiumVideoLock.tsx）
- `useSubscriptionContext`: サブスクリプション状態取得
- `canAccessContent`: アクセス判定（src/utils/premiumAccess.ts）
- `LessonCard`: カードデザイン参考（src/components/lessons/LessonCard.tsx）
- `Button`: ボタン（variant="default", size="medium"）

## ナビゲーションの暫定

以下のように今後していくのでその中でのお役立ち（ガイドとナレッジが合わさったもの）、と、質問、フィードバックのページを作っていく

#### 1. メイン (Main Navigation)

学習のコアとなる体験へのアクセス。

| 名称 | 役割定義 |

| :--- | :--- |

| **マイページ** | ユーザー個人のダッシュボード。進捗確認など。 |
| **トップ** | サービスのホーム画面。|
| **ロードマップ** | ゴール達成のためのメインルート（未経験転職、AIフローなど）。 |
| **コース** | ロードマップを構成する、単体の学習カリキュラム群。 |
| **トレーニング** | 手を動かす実践課題・ドリル。 |

#### 2. 探す (Explore / Resource)

目的のリソースを見つけるエリア。

| 名称 | 役割定義 |

| :--- | :--- |

| **探索** | 全体検索、またはキーワードからコンテンツを探す入り口。 |

| **お役立ち** | **(Oyaku-dachi)**<br>Tips、ガイド、コラムなどの「助けになる情報」の集合体。<br>動画・テキスト混合。 |

#### 3. コミュニティ (Community)

他者とのつながり、フィードバック。

| 名称 | 役割定義 |

| :--- | :--- |

| **みんなの作業部屋** | コワーキングスペース的な機能。 |

| **質問** | 疑問解決フォーラム。 |

| **フィードバック** | 制作物へのレビュー依頼。 |

| **Slackを開く** | 外部コミュニティ(Slack)へのリンク。 |

#### 4. サポート・設定 (Footer/System)

サービス利用の補助。

| 名称 | 役割定義 |

| :--- | :--- |

| **使い方** | サービスの利用ガイド。 |

| **BONOとは** | コンセプト、理念。 |

| **運営ブログ** | 運営からのお知らせ、更新情報。 |

| **設定** | アカウント設定、通知設定など。 |
