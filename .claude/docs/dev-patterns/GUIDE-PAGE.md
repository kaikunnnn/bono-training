## 📊 ナビゲーション上の位置

### グローバルナビの変更案

```
現在: [ロードマップ▼] [UX] [UI] [Figma] [ビジュアル] [キャリア] [BONOの特徴] [料金]

提案: [転職ガイド] [ロードマップ▼] [コース▼] [BONOについて] [料金]
                                    └ UI / UX / Figma / ビジュアル / キャリア
```

**転職ガイドをナビの最左（最初の項目）に置く理由:**

- 新規ユーザーが最初に見るべきものだから
- SEO的にも「UIUXデザイナー 転職」「未経験 UIデザイナー」のランディング先になる
- 「デザイナーの視点」記事がSEO経由で新規流入を生む
- ロードマップは「やると決めた人」のもの → 2番目

---

## 🎯 この構造が解決すること

### 迷い①：ロードマップとガイドの違いが曖昧になる

**→ 解決:** 明確な定義を与えました。

- **ガイド = 読んで理解するもの（記事+図解+視点）**
- **ロードマップ = 動画を見てやるもの（カリキュラム+お題）**

### 迷い②：一緒のページにすると引っかかる

**→ 解決:** 別のページにしつつ、相互リンクで自然に行き来できる設計。

### 迷い③：違うものにすればするほど違いに悩む

**→ 解決:** URL構造で物理的に分離（`/guide/` vs `/rdm/`）しつつ、ユーザーの旅の中では自然に繋がる。
Airbnbで言えば「ヘルプセンター（ガイド）」と「ホスト登録ステップ（ロードマップ）」は完全に別のものだけど、ヘルプセンター内から登録ステップに自然に誘導される ── **この関係性** です。

### 迷い④：業界トピック記事（AI時代のデザイナーの価値など）はどこに置くのか

**→ 解決:** ガイドの中に **「🔭 デザイナーの視点」** というサブカテゴリを作る。

- ブログにしない理由：BONOの文脈から切れ、消費されて終わるから
- ガイドの中に置く理由：「不安 → 安心 → だから今これを学ぶ → ロードマップ」の自然な流れが作れるから
- 独立セクションにしない理由：セクションが増えすぎるとユーザーが迷うから

### 📋 キャリアの基本（各サブページの内容設計）

#### 1. UI/UXデザイナーとは `/guide/what-is-uiux`

- UI/UXデザインの定義（Webデザインとの違い含む）
- 仕事内容の具体例（1日のスケジュール的なもの）
- 市場動向（求人数の推移、AI時代の展望）
- → **既存記事の統合先**: `become-uiux-designer-beginner-guide`

#### 2. 必要なスキルと条件 `/guide/skills-requirements`

- 3つの基礎（UIビジュアル / 情報設計 / UXデザイン）の解説
- スキルレベルの目安（どこまで出来れば転職できるか）
- 学習期間の目安
- → **既存記事の統合先**: `rdm-howtobeadesigner`, `rdm-skilllevel`, `rdm-goalreason`

#### 3. ポートフォリオの作り方 `/guide/portfolio`

- ポートフォリオが必要な理由
- 掲載すべき内容（3つの基礎×アウトプット）
- 実例紹介（転職成功者のポートフォリオ）
- 自主制作でもOKな理由
- → **既存記事の統合先**: `portfolio-01`, `portfolio-02`, `portfolio-03`

#### 4. 会社選び・面接対策 `/guide/job-search`

- 会社の種類（事業会社 vs 制作会社）
- 探し方ガイド（媒体・直接応募）
- 面接で見られるポイント
- 質問例と対策
- → **既存記事の統合先**: `how-to-find-jobs-for-beginner-designers`, `uiux-interview-tips-and-checkpoints`

#### 5. 年収・キャリアパス `/guide/career-path` ※新規

- 未経験入社の年収レンジ
- 1年目→3年目→5年目のキャリアステップ
- スペシャリスト vs マネジメント
- フリーランスという選択肢
- → **既存記事の参照**: `career-fee-fleerance`, `career-webdesign-value`

### 🔭 デザイナーの視点（★ 新カテゴリ）

**コミュニティの不安や業界トレンドに対して、BONOとしての見解・視点を提供するセクション。**

#### コンテンツの特徴

| 項目         | 説明                                                                           |
| ------------ | ------------------------------------------------------------------------------ |
| **目的**     | 業界の変化に対するデザイナーの不安を解消し、「だから今これを学ぶべき」に繋げる |
| **トーン**   | 個人ブログではなく、BONOとしての専門的な見解                                   |
| **更新頻度** | 月1〜2本程度（業界の動きやコミュニティの声に応じて）                           |
| **形式**     | テキスト記事 or 動画（YouTube向けにもなるコンテンツ）                          |
| **CTA**      | 常に「必要なスキル → ガイド」「学び方 → ロードマップ」に接続                   |
