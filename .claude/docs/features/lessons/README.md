# レッスンページ機能

**ステータス**: 実装済み（カテゴリ構造の見直し中）
**パス**: `/lessons`、`/lessons/category/:categoryId`

---

## 概要

レッスン一覧ページ。Sanityから取得したレッスンを「なりたい状態」でカテゴリ分けして表示する。

---

## 現在のカテゴリ構造

**ファイル**: `src/pages/Lessons.tsx` の `SECTIONS` 定数

| ID       | ラベル     | タイトル（なりたい状態）             | レッスン数 |
| -------- | ---------- | ------------------------------------ | ---------- |
| `visual` | UIデザイン | 魅力的なビジュアルを作れるようになる | -          |
| `ia`     | 情報設計   | 迷わない画面構造を作れるようになる   | -          |
| `ux`     | UXデザイン | ユーザー体験を設計できるようになる   | -          |
| `figma`  | Figma      | プロの道具を使いこなせるようになる   | -          |
| `career` | キャリア   | デザイナーとしてキャリアを切り開く   | -          |
| `others` | その他     | Other Topics                         | -          |

---

## カテゴリ定義フォーマット

各カテゴリは以下の構造を持つ：

```typescript
{
  id: string;              // URL用ID（例: 'visual'）
  label: string;           // タブ表示名（例: 'UIデザイン'）
  title: string;           // なりたい状態（例: '魅力的なビジュアルを作れるようになる'）
  description: string;     // カテゴリの説明文
  priorityTitles: string[]; // 優先表示するレッスンタイトル（部分一致）
  categories: string[];    // マッチングに使うカテゴリ名（Sanityのcategoryフィールド）
}
```

### マッチングロジック

1. **優先タイトルマッチング**: レッスンタイトルが `priorityTitles` に部分一致するか
2. **カテゴリマッチング**: Sanityの `category` フィールドが `categories` に部分一致するか
3. **フォールバック**: どれにもマッチしない場合は `others` へ

## 実装アプローチ案

### A案: 段階的移行

1. まず新カテゴリIDとマッチングルールを追加
2. 「その他」に落ちるレッスンを確認
3. 微調整してから旧カテゴリを削除

### B案: 一括置換

1. SECTIONSを新構造に完全置換
2. マッチングが正しいか全レッスンで検証

### 推奨: A案（段階的移行）

- 影響範囲を確認しながら進められる
- 問題があればロールバックしやすい
