## 1. プロジェクト概要

### 1.1 目的
新しいBONOサイトのトップページプロトタイプを作成する。現行サイト(bo-no.design)のコンテンツを継承しつつ、Stripeのようなモダンなグリッドレイアウトで再構成する。

### 1.2 参照
- **コンテンツ参照**: https://bo-no.design/
- **レイアウト参照**: https://stripe.com/en-jp

---

## 2. ユーザー分析

### 2.1 ターゲットユーザー
| ペルソナ | 特徴 | ニーズ |
|---------|------|--------|
| **キャリアチェンジャー** | 他業種からUI/UXデザイナーを目指す | 実践的スキル、転職成功事例 |
| **ジュニアデザイナー** | 経験1-2年、スキルアップしたい | 体系的な学習、フィードバック |
| **独学者** | 高額スクールに通えない | 手頃な価格、段階的学習 |

### 2.2 ユーザージャーニー（トップページ）
```
訪問 → 価値理解 → サービス内容把握 → 信頼性確認 → アクション（登録/詳細）
```

## 3. コンテンツ構造

### 3.1 必須セクション（優先度順）

| # | セクション | 目的 | 参照元 |
|---|-----------|------|--------|
| 1 | **ヒーロー** | 第一印象、価値提案 | BONO: メインビジュアル |
| 2 | **学習パス** | 何を学べるか一目で理解 | BONO: 学習パスウェイ |
| 3 | **実績・統計** | 信頼性の担保 | Stripe: Stats Section |
| 4 | **コース紹介** | 具体的なコンテンツ | BONO: カテゴリ別コース |
| 5 | **成功事例** | ソーシャルプルーフ | BONO: メンバーインタビュー |
| 6 | **料金プラン** | 価格の透明性 | BONO: サブスクリプション |
| 7 | **CTA** | 行動喚起 | Stripe: CTA Footer |

### 3.2 セクション詳細

#### ヒーロー
- **見出し**: 「すべての人に創造性の夜明けを」
- **サブコピー**: UI/UXデザインを体系的に学べるオンラインスクール
- **CTA**: 「無料で始める」「コースを見る」
- **ビジュアル**: デザイン作業のイメージ

#### 学習パス（グリッド）
- UI/UXデザイナーへの転職ロードマップ
- スキル別コース（ビジュアル基礎、情報設計、Figma）
- 期間表示（1ヶ月モジュール）

#### 実績・統計
- 受講者数
- コース数
- コミュニティメンバー数
- 転職成功率

#### コース紹介（グリッド）
- カテゴリ: UI, UX, Figma, キャリア
- 各36+本のビデオ
- サムネイル + タイトル + 説明

#### 成功事例
- 転職成功者のインタビュー
- Before/After（前職 → デザイナー）
- 顔写真 + コメント

#### 料金プラン
- 3つのプラン: ¥5,800〜¥15,800/月
- 機能比較表
- CTA: 「プランを選ぶ」

## 4. デザインパターン

### 4.1 Stripeスタイルのレイアウト原則

```
┌─────────────────────────────────────────────┐
│ ナビゲーション（スティッキー）               │
├─────────────────────────────────────────────┤
│ ヒーロー（フルワイド、中央揃え）            │
├─────────────────────────────────────────────┤
│ ┌─────────┐ ┌─────────┐ ┌─────────┐         │
│ │ カード1 │ │ カード2 │ │ カード3 │ グリッド │
│ └─────────┘ └─────────┘ └─────────┘         │
├─────────────────────────────────────────────┤
│    数字      数字      数字      数字       │ 統計
├─────────────────────────────────────────────┤
│ ┌─────────────────────┬──────────────────┐  │
│ │     テキスト        │    ビジュアル   │  │ 交互配置
│ └─────────────────────┴──────────────────┘  │
├─────────────────────────────────────────────┤
│              CTA セクション                 │
└─────────────────────────────────────────────┘
```

### 4.2 コンポーネント

| コンポーネント | 用途 |
|---------------|------|
| HeroSection | メインビジュアル |
| FeatureGrid | 機能・コースカード |
| StatsBar | 実績数字 |
| TestimonialCard | 成功事例 |
| PricingTable | 料金プラン |
| CTASection | 行動喚起 |

## 5. プロトタイプパターン

### パターンA: Stripe風グリッド重視
- グリッドカードを多用
- 統計を目立たせる
- クリーンでプロフェッショナル

### パターンB: ストーリーテリング重視
- 成功事例を大きく配置
- 感情に訴える構成
- 転職成功への道のりを強調

### パターンC: コンテンツ探索重視
- コース一覧を前面に
- フィルター・検索を強調
- すぐに学習を始められる印象
