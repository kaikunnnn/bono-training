# トップページ 情報設計コンセプト

**作成日**: 2025-02-25
**作成者**: Airbnb PDM視点
**目的**: 「BONOで何がデザインを鍛えられるのか」をユーザーに伝える

---

## 課題定義

### ユーザーの疑問
```
「BONOに入ったら、具体的に何ができるようになるの？」
「どんなスキルが身につくの？」
「本当に転職できるの？」
```

### 解決すべきこと
1. 学べるスキルの具体性
2. 学習の道筋（ロードマップ）
3. 結果の証明（転職実績）
4. 自分に合うかの判断材料

---

## 3案比較

| 観点 | 案A: スキルマップ | 案B: 変化ストーリー | 案C: ロードマップ |
|------|-----------------|-------------------|-----------------|
| **主役** | スキル・コンテンツ | 成功した人 | 学習の道筋 |
| **訴求** | 「これが学べる」 | 「こう変われる」 | 「こう学ぶ」 |
| **感情** | 好奇心・探索欲 | 共感・希望 | 安心・確信 |
| **向いている人** | 何を学ぶか決めたい | 転職できるか不安 | 計画的に学びたい |
| **Airbnb参考** | カテゴリ探索 | ホストストーリー | ホストになる手順 |

---

## 推奨

**案C: ロードマップ型** を第一候補として推奨

### 理由
1. **BONOの最大の強み** = 「転職実績のあるロードマップ」を直接訴求
2. **ユーザーの不安** = 「何をどう学べばいいかわからない」を解消
3. **差別化** = 他のデザインスクールにない明確な道筋

### ただし
- 案Bの「変化ストーリー」はセクションとして必ず入れる
- 案Aの「スキルマップ」は詳細ページに活用
