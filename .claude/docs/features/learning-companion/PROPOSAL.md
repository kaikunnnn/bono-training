# 企画書：学習コンパニオン機能
## 「自己学習の不安をゼロにする、BONOの学習伴走者」

**作成日**: 2026-04-17
**ステータス**: 提案中
**対象ブランチ**: feature/learning-companion（未作成）

---

## 1. 課題定義

### ユーザーが離脱する本当の理由

> 「Figmaの操作は覚えた。でも、このやり方で合っているのかわからない。不安になってきた。」

BONOのユーザーは**コンテンツの質**に不満があって辞めるのではない。
**「自分が正しく学べているか確認できない」という孤独感**が離脱の本質。

```
学習フローとユーザーの不安ポイント

[コンテンツ発見] → [学習開始] → [Quest消化] → [アウトプット] → [次へ進む]
      ✅ AI相談        ❌ 沈黙       ❌ 沈黙        ❌ 沈黙        ❌ 沈黙
  「何を学ぶ？」   「合ってる？」「理解できた？」「これでいい？」「何が次？」
```

現在の検索＋AI相談は**入口（何を学ぶか）**は解決している。
しかし**学習中〜完了後**の7割の時間はユーザーが1人で不安を抱えている。

### 課題の定量感

| 指標 | 推定値 |
|---|---|
| 「このやり方で合っているか不安」を経験するユーザー割合 | 70〜80% |
| 不安が解消されず離脱するタイミング | Quest 2〜3回目あたり |
| 1人運営でフィードバックできるキャパ | 限定的 |

## 2. ソリューション概要

### コンセプト：「AI家庭教師」が隣にいる学習体験

学習の各フェーズに、適切なタイミングでAIが介入し不安を取り除く。

```
[Quest完了]
    ↓
AIが「理解確認」を促す   ←── 学習コンパニオン（Phase 1）
    ↓
「合ってます」の確信
    ↓
学習が続く ✅
```

```
[つまずき・疑問]
    ↓
同じ疑問のスレッドを発見   ←── 質問スレッド（Phase 2）
    ↓
「みんなも同じだった」の安心感
    ↓
学習が続く ✅
```

## 3. 機能詳細

### Phase 1: AIチェックポイント 🎯
**「Quest完了後、AIが理解を確認してくれる」**

#### ユーザーストーリー
```
UIデザインを学んでいるAさん（入門者・転職志望）

Quest「タイポグラフィ」の記事を全部読んだ。
でも「フォントの対比」の概念がふんわりしている気がする。
誰かに確認してほしいけど、質問するほどでも…と思っていた。

→ 「Questを完了しました。理解確認をしますか？」というボタンが出た
→ AIに「フォントの対比を自分の言葉で説明してみてください」と聞かれた
→ 書いてみたら「その理解で合っています。補足すると〜」と返ってきた

「あ、わかってたんだ」という確信が得られた。次のQuestへ進める。
```

#### 機能仕様
- Quest完了ボタン押下後に「AI理解確認」オプションを表示
- AIがQuest内容に応じた確認質問を自動生成
  - 例）「この単元で学んだことを1文で説明すると？」
  - 例）「実際の作品でどう使いますか？」
- ユーザーが自由記述 → AIが評価＋補足フィードバック
- 「合格」「もう少し」の2値判定は出さない（圧迫感を避ける）

#### UIイメージ
```
┌─────────────────────────────────────────┐
│ ✅ Quest 2「タイポグラフィ」完了！        │
│                                         │
│  AIに理解を確認してもらいませんか？       │
│  ・回答はあなただけが見えます           │
│  ・2〜3分で完了します                   │
│                                         │
│  [AIチェックを受ける]  [スキップ]        │
└─────────────────────────────────────────┘
```

---

### Phase 2: 質問スレッド 💬
**「同じ疑問を持つ仲間がいる、という安心感」**

#### ユーザーストーリー
```
デザイン勉強中のBさん（独学3ヶ月）

「配色の60:30:10は絶対守らないといけないの？」と思ったが
Googleで調べても答えがまちまちで混乱した。

→ BONOの該当Questに質問スレッドがあった
→ 「同じ質問が17件」と表示されていた
→ AIの回答と、先輩ユーザーのコメントがあった

「あ、みんな同じとこで迷うんだ」という安心感と、
BONOのコンテキストに合った答えを得られた。
```

#### 機能仕様
- Lesson・Quest単位でスレッドを紐付け
- 投稿するとAIが最初の回答を自動生成（待ち時間ゼロ）
- 他ユーザーが「同じ疑問」ボタンで共感できる
- 運営（1人）が任意でコメント追加可能
- 匿名投稿OK（ハードルを下げる）

#### UIイメージ
```
Quest「色のきほん」
─────────────────────────────────────
よくある質問  [+ 質問する]

💬 「配色の比率って絶対60:30:10？」
   👍 同じ疑問 17人  |  AI回答あり

💬 「彩度を下げすぎるとどうなる？」
   👍 同じ疑問 8人  |  AI回答あり

💬 「補色は使いにくいのでは？」
   👍 同じ疑問 3人
─────────────────────────────────────
```

---

### Phase 3: 学習ペーストラッカー 📈
**「自分がどこまで来たか見える」**

#### 機能仕様
- Questごとの進捗をマイページで可視化
- 「〇日間ストリーク」などの継続動機
- AIが進捗に合わせて「次のアクション」を提案
- 久しぶりのログイン時に「前回の続き」を表示

## 4. 期待効果（KPI）

| KPI | 現状（推定） | 目標 | 根拠 |
|---|---|---|---|
| 30日継続率 | - | +15〜20% | 学習中の不安解消による離脱低下 |
| AIチェック実施率 | - | 40〜60% | Quest完了後の自然な動線 |
| 質問スレッド投稿率 | - | 10〜20% | 匿名・ハードル低 |
| 有料プランへの転換率 | - | +5〜10% | AI相談で価値実感後に上位機能へ |

**参考**: Duolingoはストリーク機能だけで継続率が25%向上した（公式発表）

## 5. 実装計画

### Phase 1: AIチェックポイント（2〜3日）
```
Step 1: Quest完了UIにチェックポイントボタン追加
Step 2: Questのコンテンツ情報をAIに渡す（既存api/ai-chat.tsを拡張）
Step 3: AIがQuest内容に応じた確認質問を生成するプロンプト設計
Step 4: 回答→AIフィードバックのUI実装
```
**必要なもの**: 既存のAI基盤（Groq）をそのまま活用。追加コストなし。

### Phase 2: 質問スレッド（5〜7日）
```
Step 1: Supabaseにthreads/posts/reactionsテーブル作成
Step 2: Quest・Lessonへの紐付け
Step 3: 投稿UI + AI自動回答
Step 4: 一覧表示UI
```
**必要なもの**: Supabase（既存）、AI（既存）。追加コストほぼなし。

### Phase 3: ペーストラッカー（3〜5日）
```
Step 1: 進捗データの集計ロジック
Step 2: マイページの進捗UI
Step 3: AI連携（進捗に応じた推薦）
```

## 6. リスクと対策

| リスク | 対策 |
|---|---|
| AIチェックの回答が的外れ | Questのコンテンツを詳細に渡す。プロンプトの反復改善 |
| 質問スレッドが荒れる | 匿名でも投稿前に簡易確認。問題投稿は非表示に |
| 機能が多すぎてUXが複雑化 | Phase 1のみ先行リリースし、反応を見て判断 |

---

## 7. まとめ・推薦アクション

BONOのユーザーは**「正しく学べているか確認したい」という欲求**を持っている。
これはAIがいれば24時間対応できる課題であり、1人運営でも実現可能。

**最初の一手として、Phase 1（AIチェックポイント）を推薦する。**

- 実装コスト：低（既存AI基盤の活用）
- 期待効果：高（学習中の不安を直接解消）
- リスク：低（プロンプト改善で対応可能）

プロトタイプは `/prototype/learning-companion` で確認できます。
