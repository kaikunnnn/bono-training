# Q-08: 質問投稿機能 - デザイン改善アイデア

**作成日**: 2025-02-20
**協力**: PM × Product Designer

---

## PMレビューからの主要課題

1. **投稿フローの心理的障壁が高い** - フォームが長く、入力を躊躇させる
2. **投稿後の「待ち」体験が未設計** - 回答を待つ間の不安
3. **質問の「価値」が感じられない** - コミュニティとの繋がりが薄い

---

## デザイン改善パターン 3案

### Pattern A: ステップ形式フォーム（Wizard Pattern）

**コンセプト**: 「一度に全部入力」から「一歩ずつ進む」へ

**フロー**:
```
Step 1: 何について質問？
  └─ カテゴリ選択（大きなカード形式）

Step 2: 一言で説明すると？
  └─ タイトル入力のみ

Step 3: 詳しく教えて
  └─ 質問内容（テンプレート付き）
  └─ Figmaリンク（オプション）
  └─ 参考URL（オプション）

Step 4: 確認して投稿
  └─ プレビュー表示
  └─ 投稿ボタン
```

**UI特徴**:
- 各ステップは画面の70%を使用
- プログレスバーで進捗を可視化
- 「前へ」「次へ」ボタンで自由に行き来
- Step 3のオプション項目は折りたたみ式

**心理的効果**:
- 最初のステップが簡単なので「始めやすい」
- 小さな達成感の積み重ねで「完了しやすい」
- いつでも戻れるので「安心感」がある

---

### Pattern B: 会話型フォーム（Conversational UI）

**コンセプト**: フォーム入力を「カイくんとの会話」に変える

**フロー**:
```
[カイくん] 「こんにちは！何について質問したい？」
  └─ カテゴリをチップボタンで選択

[カイくん] 「なるほど！一言で説明すると？」
  └─ タイトル入力

[カイくん] 「詳しく教えて！下のテンプレートを使うと伝わりやすいよ」
  └─ 質問内容入力

[カイくん] 「Figmaファイルがあれば共有してもらえると嬉しいな」
  └─ Figmaリンク入力（スキップ可）

[カイくん] 「これで大丈夫？」
  └─ プレビュー + 投稿ボタン
```

**UI特徴**:
- チャット形式のレイアウト
- カイくんのアバターが各質問に表示
- 入力完了するとフェードアウトして次へ
- スキップボタンでオプション項目を飛ばせる

**心理的効果**:
- カイくんとの対話感で「親しみやすい」
- 1つずつ聞かれるので「圧倒されない」
- 実際に回答してくれる人の存在を意識できる

---

### Pattern C: 投稿後の体験強化（Post-Submit Experience）

**コンセプト**: 投稿完了後の「待ち時間」を価値ある体験に変える

**投稿完了画面**:
```
┌─────────────────────────────────────────┐
│  ✅ 質問を投稿しました！                  │
│                                          │
│  📅 回答予定: 通常1-3日以内               │
│                                          │
│  [Slackで通知を受け取る] [メールで受け取る] │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  💡 回答を待つ間に...                     │
│                                          │
│  ─────────────────────────────          │
│  🔥 同じカテゴリの人気質問                │
│                                          │
│  • ポートフォリオの構成について          │
│    👀 234人が閲覧 | ✅ 回答済み           │
│                                          │
│  • Figmaのコンポーネント設計             │
│    👀 189人が閲覧 | ✅ 回答済み           │
│                                          │
│  [もっと見る →]                          │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  📊 あなたの質問                          │
│                                          │
│  「ポートフォリオのレイアウトについて」    │
│  📝 投稿日: 2025/02/20                   │
│  🔔 ステータス: 回答待ち                  │
│                                          │
│  [質問を編集] [質問を取り下げ]            │
└─────────────────────────────────────────┘
```

**マイページ「あなたの質問」セクション**:
```
┌─────────────────────────────────────────┐
│  📮 あなたの質問                          │
│                                          │
│  ┌─────────────────────────────────────┐│
│  │ 🟡 回答待ち                         ││
│  │ ポートフォリオのレイアウトについて   ││
│  │ 投稿: 2時間前                       ││
│  └─────────────────────────────────────┘│
│                                          │
│  ┌─────────────────────────────────────┐│
│  │ ✅ 回答済み（新着！）                ││
│  │ 配色の選び方について                 ││
│  │ 回答: 1日前                         ││
│  └─────────────────────────────────────┘│
└─────────────────────────────────────────┘
```

**心理的効果**:
- 回答予定時期で「不安を軽減」
- 関連質問で「待ち時間を有効活用」
- マイページで「自分の質問を追跡可能」

---

## 実装パターンの選択

### 推奨: Pattern A + Pattern C のハイブリッド

**理由**:
1. **Pattern A（ステップ形式）** は実装コストが比較的低く、即座に効果が出る
2. **Pattern C（投稿後体験）** はPMレビューで最も重要視された課題を解決
3. **Pattern B（会話型）** は魅力的だが実装コストが高く、将来の選択肢として保持

### 3つの実装パターン（/dev ページ用）

| パターン | 内容 | 特徴 |
|----------|------|------|
| **Pattern 1** | ステップ形式フォーム | 3ステップに分割、プログレスバー |
| **Pattern 2** | 投稿完了体験強化 | 回答予定表示、関連質問レコメンド |
| **Pattern 3** | 会話型フォーム | チャット風UI、カイくんと対話 |
