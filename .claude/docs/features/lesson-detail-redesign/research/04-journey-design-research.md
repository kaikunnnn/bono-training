# ジャーニーデザイン調査レポート

**作成日**: 2025-03-02
**目的**: レッスン詳細ページのジャーニー設計改善のための事例調査

---

## エグゼクティブサマリー

本調査では、Notion、Linear、Stripe、Figma、Appleの5社のオンボーディング・ストーリー設計を分析し、BONOのレッスン詳細ページに適用可能なジャーニーデザインパターンを抽出した。

### 核心的な発見

| 課題 | 解決パターン | 参考事例 |
|------|-------------|----------|
| 入口設計がない | パーソナライズWelcome + 全体地図 | Notion, Linear |
| 出口設計が弱い | 次アクションへの明確な導線 | Stripe, Duolingo |
| 「情報」止まり | ストーリーテリング + 感情設計 | Apple, Figma |

## 1. Notion: パーソナライズドオンボーディング

### 概要
Notionは、複雑なワークフローに新規ユーザーをオンボーディングする際、「情報の洪水」ではなく「会話的な誘導」を採用している。

### 学べるパターン

#### 1.1 セルフセグメンテーション
- **何をしているか**: 最初にユースケースを質問（個人/チーム/学校など）
- **なぜ効果的か**: ユーザーの文脈に合わせた体験を提供
- **BONOへの応用**: 「あなたはどんなデザイナーですか？」で学習パスをパーソナライズ

```
[Notionの質問フロー]
1. どのように使いますか？ → 個人 / チーム / 学校
2. 主な目的は？ → プロジェクト管理 / ノート / Wiki
3. チームの規模は？ → 1-5人 / 6-20人 / 20人以上
```

#### 1.2 テンプレートベースのスキャフォールディング
- **何をしているか**: 空白の画面ではなく、プリビルドのテンプレートを提供
- **なぜ効果的か**: 認知負荷と意思決定疲れを軽減
- **BONOへの応用**: レッスン開始時に「このレッスンで作るもの」のプレビューを表示

#### 1.3 プログレスインジケーター
- **何をしているか**: アカウント作成プロセスで現在位置を表示
- **なぜ効果的か**: 残りステップが見えることで完了へのモチベーション向上
- **BONOへの応用**: レッスン全体の中で「今どこにいるか」を常に表示

### 数値的成果
- オンボーディング完了時間: 約50秒（個人アカウント）
- 離脱率の低減に貢献

- [How Notion Crafts a Personalized Onboarding Experience](https://www.candu.ai/blog/how-notion-crafts-a-personalized-onboarding-experience-6-lessons-to-guide-new-users)

## 2. Linear: ハンズオン学習と映画的トランジション

### 概要
Linearは、複雑なプロダクト開発ツールを「魔法のような」体験でオンボーディングする。10以上のステップがあるにも関わらず、ユーザーは「多い」と感じない。

### 学べるパターン

#### 2.1 2段階オンボーディング構造
```
[第1セクション: Must-Do（必須）]
├── サインアップ
├── メール認証
└── ワークスペース名設定

[第2セクション: Get Started（コア体験）]
├── Part 1: 初心者向け基本操作
└── Part 2: 上級者向け機能
```

- **BONOへの応用**: レッスンを「必須学習」と「深掘り」に分離

#### 2.2 1入力/1ステップの原則
- **何をしているか**: 各画面で1つの入力のみを求める
- **なぜ効果的か**: 認知負荷を最小化し、スムーズな進行を実現
- **BONOへの応用**: コンテンツを細かく分割し、1画面1コンセプト

#### 2.3 ハンズオン体験
- **何をしているか**: 見るだけでなく、実際に操作させる
- **なぜ効果的か**: 能動的学習による定着率向上
- **BONOへの応用**: 「見るだけモード」と「実践モード」の切り替え

#### 2.4 多層的リソース提供
```
[Linearのリソース階層]
├── ライブオンボーディングセッション
├── Learning Library（動画ライブラリ）
├── Slackコミュニティ
└── ドキュメント
```

- **BONOへの応用**: レッスン完了後の次のステップを複数提示

## 3. Stripe: 開発者ファーストのドキュメント体験

### 概要
Stripeのドキュメントは「開発者体験のゴールドスタンダード」と呼ばれる。その秘密は、インタラクティブな学習体験にある。

### 学べるパターン

#### 3.1 3カラムレイアウト
```
[Stripeドキュメントの構造]
┌─────────────┬─────────────────────┬────────────────┐
│  ナビ       │   コンテンツ        │  コード例      │
│  ゲーション │                     │  （ライブ）    │
│             │                     │                │
│  • 製品     │  説明文             │  コピー可能    │
│  • 機能     │  ステップ           │  実行可能      │
│  • API      │  チュートリアル     │  ハイライト    │
└─────────────┴─────────────────────┴────────────────┘
```

- **BONOへの応用**: 説明、ビジュアル、実践の3層構造

#### 3.2 ホバーでハイライト連動
- **何をしているか**: 説明文にマウスオーバーすると、対応するコード部分がハイライト
- **なぜ効果的か**: 抽象と具体の接続を視覚化
- **BONOへの応用**: 解説とデザイン要素を連動表示

#### 3.3 Quickstart + 深掘りの構造
```
[各プロダクト領域の構造]
├── Quickstart（すぐに始める）
├── Tutorial（ステップバイステップ）
├── Guides（ユースケース別）
└── API Reference（詳細リファレンス）
```

- **BONOへの応用**: 「3分で概要」→「30分で実践」→「深掘りリソース」

#### 3.4 フリクションログ
- **何をしているか**: ユーザー視点でフローを体験し、摩擦点を記録
- **なぜ効果的か**: 実際のユーザー体験の問題を発見
- **BONOへの応用**: レッスン体験のフリクションログを定期実施

## 4. Figma: Show, Don't Tell

### 概要
Figmaは「長いチュートリアル」ではなく、「実際にやらせる」アプローチを採用。ユーザーを空白のキャンバスに直接置き、軽いオーバーレイでツールを説明する。

### 学べるパターン

#### 4.1 アニメーション付きツールチップ
```
[Figmaのツールチップ構成]
┌────────────────────────────┐
│  [機能名]                  │
│                            │
│  簡潔な説明文              │
│                            │
│  🎬 アニメーションGIF      │
│     ↑機能の動作を可視化    │
└────────────────────────────┘
```

- **なぜ効果的か**: テキストとビジュアルの両方で学習者に対応
- **BONOへの応用**: 各コンセプトに短いアニメーションを追加

#### 4.2 文脈依存の学習
- **何をしているか**: 機能を使おうとした瞬間に説明を表示
- **なぜ効果的か**: 必要な時に必要な情報のみ提供（認知負荷軽減）
- **BONOへの応用**: レッスン中の「次に何をすべきか」を状況に応じて提示

#### 4.3 デザインシステムとしてのオンボーディング
- **何をしているか**: チーム拡大時、デザインシステムがオンボーディングリソースに
- **なぜ効果的か**: 新メンバーの即戦力化
- **BONOへの応用**: 「学んだこと」を自分のデザインシステムとして蓄積

### コア哲学
> "Figma taught me much about onboarding: show, not tell."
> 「やらせることで教える」アプローチ

## 5. Apple: 製品ページのストーリーテリング

### 概要
Appleの製品ページは「情報提供」ではなく「感情的な旅」を設計している。スクロールするだけでワクワク感が生まれる。

### 学べるパターン

#### 5.1 映画的スクロール体験
```
[Appleページの構造]
┌─────────────────────────────────────┐
│  ヒーローセクション                 │ ← 感情的フック
│  大きなビジュアル + キャッチコピー  │
└─────────────────────────────────────┘
          ↓ スクロール
┌─────────────────────────────────────┐
│  機能紹介 #1                        │ ← 価値の証明
│  余白たっぷり + アニメーション      │
└─────────────────────────────────────┘
          ↓ スクロール
┌─────────────────────────────────────┐
│  機能紹介 #2                        │ ← 期待の醸成
│  3D回転 / インタラクティブ要素     │
└─────────────────────────────────────┘
          ↓ スクロール
┌─────────────────────────────────────┐
│  CTA                                │ ← 行動への誘導
│  「今すぐ購入」                     │
└─────────────────────────────────────┘
```

- **BONOへの応用**: レッスン詳細ページを「ストーリー」として構成

#### 5.2 セクション間の高コントラスト
- **何をしているか**: 各セクションで背景色やスタイルを大きく変化
- **なぜ効果的か**: 視線誘導とスクロール継続のモチベーション
- **BONOへの応用**: レッスンの各パートで視覚的な区切りを明確に

#### 5.3 余白とコンテンツの分離
- **何をしているか**: 長い文章ではなく、短いコピー + 大きなビジュアル
- **なぜ効果的か**: 読むストレスを軽減しつつ、価値を伝達
- **BONOへの応用**: 1セクション1メッセージの原則

#### 5.4 3D/没入型要素
- **何をしているか**: 製品を3D空間で探索可能に
- **なぜ効果的か**: 実際に手に取る体験に近づける
- **BONOへの応用**: デザイン成果物のインタラクティブプレビュー

### コア哲学
> "At Apple, designers use the power of storytelling to influence others, instead of just showing what they are doing."
> 「やっていること」を見せるのではなく、「ストーリー」で影響を与える

## 6. 追加調査: Duolingoの学習ジャーニー

### 概要
オンラインコースの90%以上が途中離脱するという課題に対し、Duolingoはゲーミフィケーションで解決した。

### 学べるパターン

#### 6.1 ゴール設定からの開始
- **何をしているか**: 学習開始時に自分のゴールを設定
- **なぜ効果的か**: 内発的モチベーションの喚起
- **BONOへの応用**: 「このコースを終えたらできるようになること」を最初に選択

#### 6.2 マスコットによる感情サポート
- **何をしているか**: Duoフクロウが進捗を称賛、リマインド
- **なぜ効果的か**: 人間的なつながりの感覚
- **BONOへの応用**: 進捗に応じたメッセージや称賛

#### 6.3 バイトサイズの学習
- **何をしているか**: 1レッスン5分以内で完了可能
- **なぜ効果的か**: 継続のハードルを下げる
- **BONOへの応用**: 長いレッスンを5分単位のセクションに分割

#### 6.4 ストーリーによる文脈学習
- **何をしているか**: Duolingo Storiesで物語の中で言語を学習
- **なぜ効果的か**: 文脈の中での学習は記憶定着率が高い
- **BONOへの応用**: 実際のデザインケースを使った学習コンテンツ

### 参考リンク
- [How Duolingo Gamified Language Learning](https://www.blueoceanstrategy.com/blog/duolingo/)
- [Duolingo's Intuitive Platform Guides Users Along Their Language-Learning Journey](https://www.designrush.com/best-designs/apps/duo-lingo)

## 7. ジャーニー設計の共通パターン

### 7.1 入口設計（Welcome）のパターン

| パターン | 説明 | 採用企業 |
|----------|------|----------|
| セルフセグメンテーション | ユーザーに自己申告させ、体験をカスタマイズ | Notion, Linear |
| ビジュアルプレビュー | 完成形を最初に見せる | Apple, Figma |
| クイックウィン | 最初の30秒で「できた！」体験 | Duolingo, Linear |
| パーソナライズ質問 | 2-3問で最適な学習パスを決定 | Notion, Coursera |

### 7.2 全体像の見せ方

| パターン | 説明 | 採用企業 |
|----------|------|----------|
| プログレスバー | 現在位置と残りを可視化 | 全社 |
| チェックリスト | やるべきことの一覧と達成状況 | Linear, Notion |
| マイルストーン | 大きな達成点を明示 | Duolingo |
| ステージ分け | 初級→中級→上級の段階表示 | Coursera, Linear |

### 7.3 ステップバイステップ誘導

| パターン | 説明 | 採用企業 |
|----------|------|----------|
| ツールチップ | 文脈に応じた小さなヒント | Figma, Notion |
| インタラクティブウォークスルー | 実際に操作しながら学習 | Linear, Stripe |
| スライドアウト | メインUIを邪魔せず情報提供 | Stripe |
| 1画面1タスク | 認知負荷の最小化 | Linear |

### 7.4 次のアクションへの導線

| パターン | 説明 | 採用企業 |
|----------|------|----------|
| 明確なCTA | 次にすべきことを1つだけ提示 | 全社 |
| 複数選択肢 | 学習スタイルに合わせた選択肢 | Linear, Stripe |
| Eメールナーチャリング | 次のステップをメールでリマインド | Notion |
| コミュニティ誘導 | 仲間と学ぶ場への案内 | Linear, Figma |

### 7.5 ストーリーテリング要素

| パターン | 説明 | 採用企業 |
|----------|------|----------|
| 感情的フック | ワクワク感を生むビジュアル | Apple |
| Before/After | 学ぶ前と後の変化を可視化 | - |
| 成功事例 | 実際のユーザーストーリー | Duolingo |
| なぜ？から始める | 機能ではなく価値から説明 | Apple, Stripe |

### 8.4 感情設計のポイント

#### ワクワク感を生むタイミング
1. **レッスン開始時**: 「これを学ぶとこんなことができる」のビジュアル
2. **各セクション完了時**: 小さな称賛メッセージ
3. **レッスン完了時**: 達成感の演出（アニメーション、バッジなど）
4. **コース完了時**: 特別な祝福と証明書

#### モチベーション維持の仕掛け
1. **プログレス可視化**: 常に「どこまで来たか」が分かる
2. **バイトサイズ**: 1セクション5分以内で完了可能
3. **即座のフィードバック**: クイズや確認問題で理解度チェック
4. **コミュニティ**: 同じ学習者とのつながり

---

## 9. 実装優先度

### フェーズ1: 入口の強化（即効性高）
1. レッスン導入セクションの追加
2. 完成形プレビューの表示
3. 全体地図（プログレス）の表示

### フェーズ2: 進行中体験の改善
1. セクションごとのプログレス表示
2. ミニ地図（セクション一覧）の追加
3. 前後への明確なナビゲーション

### フェーズ3: 出口設計の追加
1. 完了画面のリデザイン
2. 次のステップ選択肢の提示
3. 進捗サマリーの表示

### フェーズ4: 感情設計の追加
1. マイクロアニメーション
2. 称賛メッセージ
3. バッジ・達成システム

## 10. まとめ

### 核心的な学び

1. **「情報」を「ジャーニー」に変える**
   - 単なるコンテンツリストではなく、ユーザーが歩む「道」として設計する
   - 「今どこにいて、どこに向かうか」を常に明示

2. **認知負荷を最小化する**
   - 1画面1タスク
   - プログレッシブディスクロージャー
   - テンプレート/スキャフォールディング

3. **感情を設計する**
   - 達成感、ワクワク感、安心感
   - ストーリーテリングによる意味付け
   - 称賛とフィードバック

4. **次のアクションを明確にする**
   - 「終わり」ではなく「次への橋渡し」
   - 複数の選択肢を提示しつつ、推奨を示す
