## Sanityデータ構造

### スキーマ階層

```
❌ Course（存在しない）
✅ Lesson → Quest → Article
```

### Lesson（レッスン）の主要フィールド

| フィールド | 型 | 説明 |
|-----------|-----|------|
| title | string | レッスンタイトル |
| slug | slug | URL用スラッグ |
| description | text | 説明文 |
| iconImage | image | アイコン画像 |
| thumbnail | image | サムネイル画像 |
| category | reference | カテゴリ参照 |
| isPremium | boolean | 有料フラグ |
| quests | array[ref] | クエスト参照配列 |

### データ取得方法

```typescript
// レッスン一覧取得
import { getAllLessonsWithArticles } from '@/services/lessons';

// サムネイル画像URL生成
import { urlFor } from '@/lib/sanity';
const thumbnailUrl = urlFor(lesson.thumbnail).url();
```

## 重要な発見

### 1. コーススキーマが存在しない

Sanityには「Course」スキーマがない。ロードマップページのコースデータは：
- **オプションA**: 静的データとして定義
- **オプションB**: 新しく「Roadmap」スキーマを作成
- **オプションC**: 既存のLessonを組み合わせて動的に構築

### 2. ステップとレッスンの関係

各「ステップ」には複数の「レッスン」が含まれる。
レッスンは Sanity の Lesson スキーマと対応している。

### 3. UIデザイン入門コースは無料

他のコースは有料（¥5,800〜/月）だが、UIデザイン入門は無料。
`isPremium: false` のレッスンで構成。

## Pattern 11 コンポーネントとのマッピング

### データ構造の対応

| Pattern 11 | Webページ | Sanity |
|------------|----------|--------|
| `header.title` | ページタイトル | ロードマップ用に別途定義 |
| `header.subtitle` | コース説明文 | ロードマップ用に別途定義 |
| `header.tags` | 対象者 | ロードマップ用に別途定義 |
| `benefits.items` | 得られるもの | ロードマップ用に別途定義 |
| `steps.items` | ステップ構造 | ロードマップ用に別途定義 |
| `stepDetails.courses` | レッスンリスト | **Sanity Lesson** |

### Sanityから取得するデータ

| データ | 取得元 | フィールド |
|--------|--------|-----------|
| レッスンタイトル | Lesson | `title` |
| レッスン説明 | Lesson | `description` |
| サムネイル | Lesson | `thumbnail` or `thumbnailUrl` |
| レッスンURL | Lesson | `slug.current` → `/lessons/{slug}` |
| 有料フラグ | Lesson | `isPremium` |

### 静的に定義するデータ

| データ | 理由 |
|--------|------|
| コースタイトル | Sanityに存在しない |
| コース説明文 | Sanityに存在しない |
| 対象者タグ | Sanityに存在しない |
| 得られるもの | Sanityに存在しない |
| ステップ構造 | Sanityに存在しない |
| ステップとレッスンの紐付け | Sanityに存在しない |

## 実装方針

### 推奨: ハイブリッドアプローチ

```typescript
// 静的コースデータ
const courseConfig = {
  'uivisual-course': {
    title: 'UIビジュアル入門コース',
    subtitle: '...',
    tags: ['未経験', '...'],
    benefits: ['...'],
    steps: [
      {
        number: '01',
        title: '上達するデザインの進め方',
        goal: '...',
        duration: '3〜7日',
        // Sanityから取得するレッスンのslug
        lessonSlugs: ['ui-design-cycle', 'sense-technique']
      }
    ]
  }
};

// Sanityからレッスン詳細を取得
const lessons = await getLessonsBySlug(lessonSlugs);
```

### メリット

1. コース構造は柔軟に変更可能（静的データ）
2. レッスン詳細は常に最新（Sanity連携）
3. 既存のSanityスキーマ変更不要
