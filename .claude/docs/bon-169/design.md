# BON-169: UX設計 — design.md

## コンセプト

「Service as a Software」の考え方を適用。BONOはレッスン動画（ツール）を売るのではなく、「UIUXデザイナーになる」「転職する」「AIを活用できるようになる」というアウトカムを売る。

ユーザーは「何をすべきかわからない」状態で来る。ガイドが道標を示し、トピックが具体的なスキル習得の場を提供し、プロセスが体系的な全体像を見せる。

---

## Phase 1: ガイド（/guides）

### 役割

「迷っている人」の入口。ゴール別に「何をどの順番で学ぶべきか」を示す道標。
ロードマップ（長期的な学習パス）と関連記事（課題別のピンポイント解説）を内包する。

### ユーザー動線

```
ユーザーの状態: 「転職したいけど何から始めればいい？」
  ↓
/guides（一覧）— 自分のゴールに合うガイドを選ぶ
  ↓
/guides/career-change（詳細）— 全体像を理解する
  ├── ロードマップで学習ステップを確認
  │     └── 各ステップ → /topics/[slug] へ（課金ポイント）
  └── 関連記事で個別の疑問を解決
```

### /guides — 一覧ページ

**目的:** ユーザーが自分のゴールに合ったガイドを見つける

**構成:**
```

│  PageHeader                              │

│  description: "目標に合ったガイドを選んで  │
│  デザインの学習を始めよう"                  │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│  ガイドカード（3列グリッド）                │
│                                          │
│  ┌──────┐  ┌──────┐  ┌──────┐          │
│  │ 🚀   │  │ 🤖   │  │ 💼   │          │
│  │転職   │  │AI活用 │  │1人   │          │
│  │ガイド │  │ガイド │  │デザ  │          │
```

**カード要素:**
- 絵文字アイコン（大きめ）
- ガイドタイトル
- 1行の説明文
- 「ロードマップ X個 / 記事 Y件」のメタ情報
- ホバーエフェクト（既存HoverCardパターン）
