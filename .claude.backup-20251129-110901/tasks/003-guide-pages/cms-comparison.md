# CMS検討: Ghost vs Markdown

## 結論

**推奨: Markdown方式（現行システムの踏襲）**

理由:
1. 既存の `/training` と同じ仕組みを活用できる
2. 開発コストが低い（既存コードの再利用）
3. インフラコスト不要
4. Git でバージョン管理が自然にできる
5. コンテンツ量がまだ少ない段階では過剰投資

## 比較表

| 項目 | Markdown方式 | Ghost CMS |
|------|-------------|-----------|
| **初期開発コスト** | ✅ 低（既存実装を再利用） | ❌ 高（新規構築必要） |
| **運用コスト** | ✅ 無料 | ❌ 月額$9〜（セルフホスト時は$0） |
| **編集の簡単さ** | ⚠️ テキストエディタ | ✅ GUI編集 |
| **バージョン管理** | ✅ Git標準 | ⚠️ 別途プラグイン必要 |
| **複数人編集** | ⚠️ Git操作必要 | ✅ 簡単 |
| **デプロイ** | ✅ 自動（静的ファイル） | ⚠️ 別途API構築 |
| **カスタマイズ性** | ✅ 完全制御 | ⚠️ 制限あり |
| **パフォーマンス** | ✅ 高速（静的） | ⚠️ API依存 |
| **検索機能** | ⚠️ 自前実装必要 | ✅ 標準搭載 |
| **メディア管理** | ⚠️ 手動 | ✅ 自動管理 |

## Markdown方式の詳細

### メリット

✅ **開発効率**
- 既存の `/training` 実装パターンを100%再利用可能
- `src/hooks/useTrainingCache.ts` と同様の hooks 作成
- `src/lib/trainingCache.ts` のガイド版を作成
- コンポーネントも流用可能

✅ **運用コスト**
- インフラ費用 $0
- メンテナンスコスト最小

✅ **バージョン管理**
- Git でコミット履歴が残る
- コードレビューが可能
- ロールバックが簡単

✅ **デプロイ**
- ビルド時に静的ファイル生成
- CDN配信で高速
- サーバー不要

### デメリット

⚠️ **編集体験**
- テキストエディタでMarkdownを直接編集
- プレビューは別途ツールが必要
- 非エンジニアには難しい可能性

⚠️ **メディア管理**
- 画像は手動で `public/assets/guide/` に配置
- リサイズや最適化は手動

⚠️ **検索機能**
- フロントエンドで実装必要
- または Algolia などの外部サービス利用

### 実装例

```typescript
// content/guide/career/job-change-roadmap/index.md
---
title: "転職を成功させるロードマップ"
category: "career"
tags: ["転職", "キャリア"]
---

# 転職ロードマップ

本文...
```

## Ghost CMS の詳細

### メリット

✅ **編集体験**
- リッチなGUIエディタ
- リアルタイムプレビュー
- メディアアップロードが簡単

✅ **複数人編集**
- ユーザー管理機能
- 権限設定
- コメント機能

✅ **機能充実**
- 検索機能標準搭載
- タグ・カテゴリ管理
- SEO最適化
- メールニュースレター

### デメリット

❌ **初期開発コスト**
- Ghost APIとの連携実装
- 認証・認可の実装
- 既存システムとの統合

❌ **運用コスト**
- セルフホスト: サーバー費用 + メンテナンス
- Ghost Pro: 月額 $9〜$199
- データベース管理

❌ **インフラ複雑化**
- Ghostサーバーの運用
- API エンドポイントの管理
- データベースのバックアップ

❌ **パフォーマンス**
- API呼び出しのオーバーヘッド
- キャッシュ戦略が必要

### アーキテクチャ例

```
[Frontend (React)]
    ↓ REST API
[Ghost CMS]
    ↓
[MySQL Database]
```

## 段階的アプローチの提案

### フェーズ1: Markdown で始める（今すぐ）

- 初期開発コストが低い
- 既存システムとの親和性が高い
- すぐに記事を公開できる

### フェーズ2: コンテンツが増えたら検討（将来）

記事数が **50本以上** になったら Ghost を検討：
- 編集者が複数人必要になった
- 非エンジニアが編集する必要が出た
- 高度な検索機能が必要になった
- メールニュースレター配信が必要になった

## 推奨: Markdown方式で開始

現時点では以下の理由から **Markdown方式** を推奨します：

1. ✅ コンテンツ量がまだ少ない（これから増やす段階）
2. ✅ 既存の `/training` 実装を活用できる
3. ✅ 開発コストが圧倒的に低い
4. ✅ 運用コストが $0
5. ✅ 必要になったら Ghost に移行可能（データ移行は簡単）

## 次のステップ

1. Markdown方式で `/guide` ページを実装
2. サンプル記事を3-5本作成
3. 運用してみてニーズを確認
4. 将来的に必要であれば Ghost に移行を検討
