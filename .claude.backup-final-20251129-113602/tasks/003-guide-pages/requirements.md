# 要件定義

## 何をやるのか

`/guide` ページを作成し、記事コンテンツベースの学習ガイドを管理・表示する機能を実装する。

## UI/UX 上の課題

現在、学習に関する体系的なガイド記事を掲載する場所がない。
ユーザーが「転職するにはどうすれば良いか」「AIでデザイナー職はどう変わるのか」「良い勉強法とは」などの疑問に答える場所が必要。

## 機能要件

### 基本機能

1. **ガイド一覧ページ (`/guide`)**
   - カテゴリ別にガイド記事を表示
   - 各記事のサムネイル、タイトル、概要を表示
   - カテゴリフィルター機能

2. **ガイド詳細ページ (`/guide/:slug`)**
   - Markdown形式の記事コンテンツを表示
   - 目次（Table of Contents）
   - 関連記事の表示
   - カテゴリ/タグ表示

3. **Wiki風一覧表示**
   - 特定のテーマに関する記事をまとめたページ
   - カテゴリページ (`/guide/category/:category`)

### コンテンツのカテゴリ例

- **キャリア**
  - 転職ロードマップ
  - デザイナーのキャリアパス

- **学習方法**
  - 良い勉強法・悪い勉強法
  - デザインの学び方

- **業界動向**
  - AIでデザイナー職がどう変わるのか
  - デザインツールの選び方

### データ管理方法の検討

**選択肢:**

1. **既存のMarkdown方式を踏襲（推奨）**
   - ✅ `content/guide/` フォルダに Markdown ファイルで管理
   - ✅ Frontmatter でメタデータ管理
   - ✅ Git でバージョン管理
   - ✅ 既存の `/training` と同じ仕組みを活用
   - ✅ 追加のCMS不要

2. **Ghost CMS を導入**
   - ❌ 別途CMS基盤の構築・運用が必要
   - ❌ インフラコストの増加
   - ✅ GUI での編集が可能
   - ✅ 複数人での編集がしやすい

**推奨: Markdown方式**
- 既存の `/training` の実装パターンを活用できる
- 開発コストが低い
- Git でバージョン管理できる

## 非機能要件

- レスポンシブ対応
- SEO対応（メタタグ、OGP）
- ページ遷移の高速化
- アクセシビリティ対応
