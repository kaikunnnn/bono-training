# 情報アーキテクチャの設計

## 基本情報

**作業時間**: 50分  
**難易度**: 中級  
**前タスクからの継続**: 要件分析の結果を活用

このタスクでは、前タスクで明確化した要件をもとに、効果的な商品カタログの情報構造を設計します。

## 情報アーキテクチャの基礎

### 情報の整理と分類の原則

1. **論理的分類**: 商品属性による客観的な整理
2. **ユーザー中心分類**: 利用者の期待に沿った分類
3. **タスク指向分類**: 目的達成を支援する分類

### ユーザーのメンタルモデルとの整合性

ユーザーが頭の中で思い描く商品の分類方法と、実際のカテゴリ設計を一致させることが重要です。

### カテゴリ設計の基本パターン

- **階層型**: トップダウンの分類構造
- **ファセット型**: 複数の属性による多軸分類  
- **ハイブリッド型**: 階層とファセットの組み合わせ

## ECサイト特有の考慮点

### 商品属性による分類方法

書籍ECサイトの場合の主要属性：
- **ジャンル**: 文芸、ビジネス、技術書など
- **形態**: 単行本、文庫本、電子書籍など
- **対象**: 一般、専門家、学生など
- **価格帯**: 〜1,000円、1,000〜3,000円など

### 検索とブラウジングの使い分け

- **検索主体**: 目的買いユーザー向けの詳細検索機能
- **ブラウジング主体**: 発見型ユーザー向けのカテゴリナビゲーション

### モバイルファーストの観点

スマートフォンでの操作性を最優先に考慮した情報階層の設計が必要です。

# プレミアムコンテンツ: 実践的な設計手法

## カテゴリ設計ワークショップ

### カードソーティング手法の実践

**目的**: ユーザーのメンタルモデルに沿ったカテゴリ構造の発見

**実施手順**:
1. **準備フェーズ**
   - 代表的な書籍100冊をカード化
   - 多様なユーザー5-8名を招集
   - オンライン/オフラインツールの準備

2. **実施フェーズ**
   - オープンカードソート（自由分類）
   - クローズドカードソート（既存カテゴリでの分類）
   - ハイブリッドカードソート（追加・変更可能）

3. **分析フェーズ**
   - 分類パターンの共通点抽出
   - 異なる意見の背景理解
   - 最適なカテゴリ構造の決定

### ツリーテスト設計方法

**目的**: 設計したカテゴリ構造の使いやすさ検証

**テストシナリオ例**:
- 「プログラミング初心者向けの本を探してください」
- 「ビジネス書の新刊を見つけてください」
- 「1,500円以下のマンガを探してください」

**評価指標**:
- タスク成功率
- 完了時間
- 最初のクリック率
- 迷いの発生箇所

### A/Bテスト可能な設計の考え方

複数のカテゴリ構造案を用意し、実際のユーザー行動で検証できるよう設計します。

**テスト要素**:
- カテゴリの命名
- 階層の深さ
- メガメニューの構成

## ナビゲーション最適化

### メガメニューの効果的な使い方

**設計原則**:
- **視覚的階層**: 重要なカテゴリを大きく表示
- **バランス**: 各列の項目数を均等に配置
- **プレビュー**: 人気商品の画像表示

**レイアウトパターン**:
```
[メインカテゴリ]     [サブカテゴリ]        [プロモーション]
文芸・小説          ├ 純文学              ┌─────────────┐
├ 日本文学          ├ 時代小説            │  今週の特集     │
├ 海外文学          ├ ミステリー          │  ─────────    │
├ SF・ファンタジー   └ ライトノベル        │ [商品画像]      │
                                          └─────────────┘
```

### ブレッドクラムの設計指針

**種類別活用法**:
- **場所型**: カテゴリ階層を表示
- **経路型**: ユーザーの実際の移動履歴
- **属性型**: 選択したフィルタ条件

### フィルタとソート機能の配置

**優先度の高い位置**:
1. ページ上部の横並び配置
2. サイドバーの固定位置
3. フローティングボタン（モバイル）

## 実装を意識した設計

### CMS構造との連携

管理画面での商品登録・更新を考慮したカテゴリ設計：

**管理しやすい構造**:
- 明確な分類基準
- 重複のない分類
- 将来の拡張性

### SEO観点でのURL設計

検索エンジン最適化を考慮したURL構造：

**Good例**:
```
/books/business/management/
/books/fiction/mystery/
```

**Bad例**:
```
/category?id=123&sub=456
/books/cat1/subcat2/
```

### 運用性を考慮したカテゴリ管理

**運用ルールの策定**:
- 新商品のカテゴリ判定基準
- カテゴリ変更時の影響範囲
- 季節やトレンドへの対応方法
