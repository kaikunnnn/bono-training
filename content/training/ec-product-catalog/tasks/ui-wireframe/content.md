# UIワイヤーフレーム作成

## 基本情報

**作業時間**: 40分  
**難易度**: 中級  
**前タスクからの継続**: 情報アーキテクチャ設計の結果を画面設計に反映

このタスクでは、前タスクで設計した情報構造をもとに、実際のユーザーインターフェースのワイヤーフレームを作成します。

## ワイヤーフレーム設計の基礎

### 設計前の準備

1. **ユーザーフロー確認**: 主要な操作手順の整理
2. **コンテンツ要素整理**: 表示すべき情報の優先度付け
3. **制約条件確認**: 技術的制約・ブランド制約の把握

### レイアウトの基本原則

- **視線の流れ**: Z型・F型の読み取りパターンを活用
- **情報のグルーピング**: 関連する要素をまとめて配置
- **階層の表現**: サイズ・色・位置による重要度の表現

## 商品一覧ページの設計

### 必須要素の配置

**上部エリア**:
- パンくずナビゲーション
- ページタイトル・件数表示
- ソート・表示件数変更

**メインエリア**:
- 商品カード（画像・タイトル・価格・評価）
- フィルタサイドバー
- ページネーション

**商品カード設計**:
```
┌─────────────────┐
│  [商品画像]      │
│                 │
├─────────────────┤
│ 商品タイトル     │
│ 著者名          │
│ ★★★★☆ (4.2)   │
│ ¥1,980 [カート] │
└─────────────────┘
```

### フィルタ機能の設計

**サイドバー型（PC）**:
- カテゴリ
- 価格帯
- 評価
- 出版年
- 在庫状況

**モーダル型（モバイル）**:
- フィルタボタンでモーダル表示
- 選択した条件をタグで表示
- クリアボタンで一括解除

## 検索結果ページの設計

### 検索体験の最適化

**検索ボックス周辺**:
- 検索キーワードの再表示
- 修正候補の提示
- 関連キーワードの提案

**結果表示エリア**:
- ヒット件数と検索時間
- 「もしかして」機能
- 結果0件時の代替提案

**検索結果の表現**:
- 検索キーワードのハイライト
- 関連度による並び順
- 商品説明文の抜粋表示

## レスポンシブ対応設計

### ブレークポイント設計

- **デスクトップ** (1200px以上): 3-4列グリッド
- **タブレット** (768-1199px): 2-3列グリッド  
- **スマートフォン** (767px以下): 1-2列グリッド

### モバイル特有の配慮

**タッチ操作への対応**:
- ボタンサイズ最小44px
- タップ可能エリアの明確化
- スワイプジェスチャーの活用

**画面表示の最適化**:
- 重要情報の上部配置
- スクロール量の最小化
- 画像の遅延読み込み

## アクセシビリティ考慮

### キーボード操作対応

- Tabキーでの順次移動
- Enterキーでの決定操作
- Escキーでのモーダル閉じる

### スクリーンリーダー対応

- 適切なheading構造
- 画像のalt属性設定
- フォーム要素のlabel設定

## パフォーマンス考慮

### 読み込み速度最適化

**画像最適化**:
- 適切なファイル形式選択
- 遅延読み込み（Lazy Loading）
- レスポンシブイメージ

**データ取得最適化**:
- 必要最小限の情報取得
- キャッシュ機能の活用
- ページネーション設計

## 実装ガイドライン

### 技術的実装ポイント

**CSS設計**:
- BEM記法でのクラス命名
- レスポンシブ対応のメディアクエリ
- フレックスボックス・グリッドレイアウト

**JavaScript実装**:
- フィルタ機能のリアルタイム更新
- 無限スクロール/ページネーション
- 検索候補のオートコンプリート

## アクションステップ

### ステップ1: PC版ワイヤーフレーム作成（20分）
Figmaで商品一覧ページのワイヤーフレームを作成しましょう。

### ステップ2: モバイル版ワイヤーフレーム作成（15分）
スマートフォン向けのレイアウトを設計しましょう。

### ステップ3: ユーザビリティチェック（5分）
作成したワイヤーフレームのユーザビリティを確認しましょう。
