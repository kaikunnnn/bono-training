# デザインの進め方の基本フロー

良いデザインを作るためには、正しいプロセスを踏むことが不可欠です。
プロのデザイナーが実践している、標準的なワークフロー（ダブルダイヤモンドモデル）について解説します。

## 1. 発見 (Discover)

まずは「何を作るべきか」「誰のために作るのか」を明らかにします。

- **ヒアリング**: クライアントやステークホルダーへの聞き取り。
- **リサーチ**: 競合調査、ユーザーインタビュー。
- **課題の洗い出し**: 現状の問題点を列挙します。

## 2. 定義 (Define)

発見した課題の中から、解決すべき本質的な問題を特定します。

- **ペルソナ設定**: ターゲットユーザー像を明確にします。
- **要件定義**: 機能要件、非機能要件をまとめます。
- **コンセプト策定**: デザインの方向性を決めます。

## 3. 展開 (Develop)

解決策を形にしていきます。

- **情報設計 (IA)**: サイトマップ、ワイヤーフレームの作成。
- **プロトタイピング**: 画面遷移を含む動的なモデルの作成。
- **ビジュアルデザイン**: 色、フォント、レイアウトの詳細設計。

## 4. 提供 (Deliver)

作成したデザインを実装可能な形にして届けます。

- **ユーザーテスト**: 実際に使ってもらい、問題がないか検証します。
- **修正**: テスト結果に基づいてブラッシュアップします。
- **実装連携**: エンジニアへのハンドオフ（デザインスペックの共有）。

## まとめ

デザインは「ひらめき」ではなく「積み上げ」で作られます。
この4つのフェーズを丁寧に進めることで、根拠のある強いデザインを生み出すことができます。
