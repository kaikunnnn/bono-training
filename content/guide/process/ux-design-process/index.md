# 独りよがりなデザインにならないために

どれだけ美しいUIを作っても、ユーザーにとって使いにくければ意味がありません。
「作り手の論理」ではなく「使い手のメンタルモデル」に合わせたUIを作るための手法を解説します。

## 1. コンテキスト（利用状況）の理解

ユーザーはいつ、どこで、どんな気持ちでそのプロダクトを使うのでしょうか？

- **移動中？家でゆっくり？**: ボタンの大きさや情報量が変わります。
- **緊急時？暇つぶし？**: 操作のスピード感や導線が変わります。

## 2. メンタルモデルとの整合

ユーザーは過去の経験から「こうすればこうなるはずだ」という予測（メンタルモデル）を持っています。

- **慣れ親しんだUIパターンを使う**: 独自の操作方法は学習コストが高いです。
- **期待通りに動く**: ボタンを押したらフィードバックがある、スクロールしたら続きが見えるなど。

## 3. オブジェクト指向UI (OOUI)

タスク（やること）ではなく、オブジェクト（もの）を起点にUIを設計する手法です。

- **名詞 → 動詞**: 「メール（名詞）を選んでから、削除（動詞）する」
- **モードレス**: ユーザーが自由に操作の順序を選べるようにする。

## 4. プロトタイピングによる検証

静止画では分からない「操作感」を確認します。

- **画面遷移の違和感はないか？**
- **アニメーションは適切か？**
- **指の届く範囲にボタンがあるか？**

## まとめ

ユーザーに合致したUI体験とは、「ユーザーが何も考えずに目的を達成できる」状態のことです。
常に「ユーザーならどう思うか？」を問いかけながら設計を進めましょう。
