# SaaSのUIは「No UI」に向かうのか

AIの進化により、チャット形式（自然言語）で操作できるインターフェースが増えています。
これにより、従来の「ボタンやフォームが並ぶ管理画面」は不要になるのでしょうか？

## 複雑なタスクにはGUIが必要

「来週の予定教えて」のような単純なタスクはチャットUI（CUI）が最適です。
しかし、複雑な業務アプリケーション（SaaS）において、全ての操作をチャットで行うのは現実的ではありません。

- **一覧性**: 多くのデータを俯瞰して見るには、表形式のUIが優れています。
- **操作効率**: 決まった手順を繰り返す場合、ボタンをクリックする方が文字を打つより早いです。
- **メンタルモデル**: 視覚的な操作盤があることで、システムの構造を理解しやすくなります。

## ハイブリッドな未来

これからは、従来のGUI（Graphical User Interface）と、AIによるLUI（Language User Interface）が融合した形が主流になるでしょう。

- **Co-pilot型**: 画面の横にAIアシスタントがいて、サポートしてくれる。
- **動的UI**: ユーザーのやりたいことに合わせて、必要なUIパーツだけが生成される。

UIデザイナーは、「いつGUIを使い、いつAIに任せるか」という、インターフェースの最適解を設計する役割を担うことになります。
